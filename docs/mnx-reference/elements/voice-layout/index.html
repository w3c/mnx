The <voice-layout> element

Parent element: <staff-layout>

The <voice-layout> element lets you combine multiple <part>s into a single voice of a <staff-layout>. This is useful for complex orchestral scores, where individual parts are sometimes placed on multiple staves but sometimes placed on a single staff.

Content

<part-layout> (One or more times)

Attributes

Name Type Required? Description
label String No An optional name for this voice. If provided, this should be rendered to the left of the staff, vertically positioned such that it aligns with the voice position (either above center or below center).

Generally we recommend using "labelref" instead of "label", to avoid duplication of data within an MNX document.
labelref Staff label type No If provided, this specifies that the corresponding part's name (or short-name) should be rendered to the left of the staff.

By using this instead of the "label" attribute, you can avoid needing to duplicate a part's name in an MNX document.

If this <voice-layout> contains multiple <part-layout> elements, the first <part-layout> child should be used.
stem Voice stem direction No The stem direction to use for all notes within this <voice-layout>. Default is "part", which means to use stem directions as defined in the first child <part-layout>.

If this value is "up", "down" or "float", then stem directions in the underlying <part> are ignored within this <voice-layout>.
