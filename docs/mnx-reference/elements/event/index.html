The <event> element

Parent elements: <grace>, <sequence>, <tuplet>

The <event> element represents a discrete period of time during which one or more notes are performed, or in which a rest occurs.

Content

In any order
  • <note> (Zero or more times)
  • <rest> (Optional)
  • <slur> (Zero or more times)

Attributes

Name Type Required? Description
color Simple color No The color to be used when rendering notational objects in this event.
duration Note value quantity No The actual, performed duration of the event, if different from the notated value given by "value".

Note that this is specified as a note value quantity, rather than a rhythmic duration, because the former is less constrained. It may be any desired multiple of a note value.

The duration attribute not only alters the performance of the event, but determines the amount of time the event takes up in the measure, affecting the location of subsequent events in the containing sequence. This can change the layout of the measure, as subsequent <event>s will occur earlier or later as a result and be positioned accordingly. It also affects the validation of the measure for metrical correctness.

This attribute is unlikely to be frequently encountered, but it's needed to handle cases in which composers use notated values that are not interpreted literally.
id Event ID No
measure Yes or no No Whether this event is a whole-measure event — meaning its duration lasts the entire measure. This is used, for example, for rests that last an entire measure.

If this value is set to "yes", then this event may not specify "value" and this event must either be empty or contain exactly one <rest>.
orient Orientation No A specific orientation for this event. If not provided, the orientation is inherited from any ancestor element that specified it. If no ancestor did so, the orientation is determined automatically according to the implementation’s rendering rules.
staff Staff index No A specific staff index for this event. If not provided, the value is inherited from any ancestor element that specified it. If no ancestor did so, the value is determined automatically according to the implementation’s rendering rules.

The primary purpose of this attribute is to override a default staff assignment at the <sequence> level — as in cross-staff keyboard notation.
stem-direction Stem direction No Whether the stem of the note(s) in this event is up or down. If not provided, this is left to consuming software to decide automatically.
value Rhythmic duration No The duration of this event. This is required unless this event has measure="yes".

Examples

This element is used in the following examples:

Accidentals, Beams, Beams (across barlines), Beams (hooks), Beams (secondary beam breaks), Beams (with inner grace notes), Dotted notes (augmentation dots), Jumps (D.S. al Fine), Jumps (Dal Segno), Key signatures, Multiple voices, Octave shifts (8va), Parts, Repeats, Repeats (more than once repeated), Repeats (with alternate endings, advanced), Repeats (with alternate endings, simple), Repeats (with implied start repeat), Slurs, Slurs (for chords), Slurs (incomplete slurs), Slurs (targeting specific notes), Three-note chord and half rest, Ties, Time signatures, Tuplets, Two-bar C major scale, “Hello world”
