The <score> element

Parent element: <mnx>

The <score> element represents a particular rendering of the music, with a defined subset of which parts are included, how they're visually arranged, where system breaks happen and where page breaks happen.

An MNX document can contain multiple <score> elements. For example, you can use this to define a "full score" vs. individual parts, within a single MNX file.

MNX documents are not required to contain a <score> element. If <score> isn't provided, the default rendering behavior is to include all parts — in the same order in which the <part> elements appear in the document — with line breaks and page breaks determined automatically by the consuming software.

A <score> element also is not required to contain any <page> elements. If <page> elements aren't provided, page breaks are to be determined automatically by the consuming software.

Content

In any order
  • <page> (Zero or more times)
  • <style> (Zero or more times)

Attributes

Name Type Required? Description
name String Yes The score name, suitable for display to people viewing or selecting this particular score within music notation software.

Examples: "Full score", "Flute 1 part".
layout System layout ID No An optional layout to use for all systems in this score. If provided, this should be the ID of a <system-layout> element in the same MNX document.

Note that individual <system> elements within this <score> can optionally override this.
