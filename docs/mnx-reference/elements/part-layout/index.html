The <part-layout> element

Parent elements: <staff-layout>, <voice-layout>

The <part-layout> element is the way to specify which <part>s are included in a given <staff-layout> or <voice-layout>.

Content

Always empty.

Attributes

Name Type Required? Description
part Part ID Yes A reference to the appropriate <part>.
part-staff Staff index No Optional staff index within the given <part>. For example, this lets you filter the <part> to only include the notation data with staff=1 (the top staff of a grand staff).
part-voice Voice ID No Optional voice ID within the given <part>. This allows you to filter the associated <part>'s <sequence> data by voice.

If this is not provided, the <part-layout> will use all the notation data from the <part> (i.e., applying no filter).
stem Part stem direction No The stem direction to use for all notes within the associated <part>. Default is "float", meaning the stem directions should be determined algorithmically.

Examples

This element is used in the following examples:

Multiple layouts, Orchestral layout, Organ layout, System layouts
