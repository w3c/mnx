The <measure> element (part)

Parent element: <part>

The <measure> element, when used within <part>, is the place to encode the musical information for a given measure in a given part. This constitutes the majority of musical content in an MNX document.

(See also <measure> when used within <global>.)

Content

In any order
  • <directions> (Optional)
  • <sequence> (One or more times)

Attributes

Name Type Required? Description
barline Barline No The barline drawn at the end of this measure. If not provided, the barline should be interpreted as follows:

* If the measure is the last in the document, use "light-heavy".
* Otherwise, use "regular".
index Measure number No The one-based index of this measure within the document's score order. The default value is 1 for the first <measure> in the <part>. For all other measures, the default value is the value of the previous <measure> plus 1.

This is used within the MNX document in cases when content needs to refer to a specific measure.
number Measure number No A visual label for the measure. It is not required to be unique in the document. If omitted, its value is the same as index.

Examples

This element is used in the following examples:

Accidentals, Beams, Beams (across barlines), Beams (hooks), Beams (secondary beam breaks), Beams (with inner grace notes), Dotted notes (augmentation dots), Grace note, Grace notes (beamed), Jumps (D.S. al Fine), Jumps (Dal Segno), Key signatures, Multiple voices, Octave shifts (8va), Parts, Repeats, Repeats (more than once repeated), Repeats (with alternate endings, advanced), Repeats (with alternate endings, simple), Repeats (with implied start repeat), Slurs, Slurs (for chords), Slurs (incomplete slurs), Slurs (targeting specific notes), Three-note chord and half rest, Ties, Time signatures, Tuplets, Two-bar C major scale, “Hello world”
