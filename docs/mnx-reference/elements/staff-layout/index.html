The <staff-layout> element

Parent elements: <group-layout>, <system-layout>

The <staff-layout> element describes the visual display of a single staff within a <system-layout>.

Generally this corresponds to a single <part>, but MNX supports encoding multiple parts in the same staff â€” as might be used, for example, when combining soprano and alto vocal parts into a single stave.

You must have at least one child <part-layout> or <voice-layout> element, to specify which part(s) are displayed within this staff layout.

Content

In any order
  • <part-layout> (Zero or more times)
  • <voice-layout> (Zero or more times)

Attributes

Name Type Required? Description
label String No An optional name for this staff. If provided, this should be rendered to the left of the staff.

Generally we recommend using "labelref" instead of "label", to avoid duplication of data within an MNX document.
labelref Staff label type No If provided, this specifies that the corresponding part's name (or short-name) should be rendered to the left of the staff.

By using this instead of the "label" attribute, you can avoid needing to duplicate a part's name in an MNX document.
symbol Grouping symbol No The symbol to use along the left vertical edge of this staff. If omitted, the default value is "none".

Note that the parent <staff-group> might also define its own "symbol" attribute.
